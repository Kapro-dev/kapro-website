# PromotionSources

<ConceptDiagram id="promotion-sources" />

`PromotionSource` defines the application units Kapro can promote.

<div class="kapro-lesson">
  <img src="/img/logo.png" alt="Kapro mascot" />
  <div>
    <strong>A PromotionSource is the menu of movable app parts.</strong>
    <p>It tells Kapro which units exist and exactly where each unit's version lives in Argo, Flux, Helm, Kustomize, or another Git-native file.</p>
  </div>
</div>

It is the source/app-unit contract for both generated greenfield layouts and
brownfield Argo or Flux layouts discovered from existing repositories.

## Shape

```yaml
apiVersion: kapro.io/v1alpha1
kind: PromotionSource
metadata:
  name: checkout
spec:
  backendRef: argo
  units:
    - name: api
      backendKind: ArgoApplicationSource
      namespace: argocd
      sourcePath: argocd/applications/api.yaml
      versionField: spec.source.targetRevision
```

`backendRef` points to the usual `BackendProfile`. Each `unit` is a stable
deployable thing that a `PromotionRun` can version.

## Greenfield Use

In greenfield mode, a `PromotionSource` can describe generated units, chart
sources, defaults, overrides, namespaces, waves, and dependencies.

That lets `kapro init` create a starter promotion repository for a new team.

## Brownfield Use

In brownfield mode, a `PromotionSource` maps existing backend-native objects:

| Backend | Example unit mapping |
|---|---|
| Argo Application | `spec.source.targetRevision` |
| Argo ApplicationSet Git file generator | JSON or YAML generator input field |
| Flux GitRepository | `spec.ref.tag`, `semver`, or `digest` |
| Flux OCIRepository | `spec.ref.tag`, `semver`, or `digest` |
| Flux HelmRelease | chart version or reviewed image tag field |
| Kustomize file | `images[].newTag` |

Kapro changes only the mapped version field. The backend still owns sync,
credentials, drift correction, health, hooks, and local rollout.

## Generated By Discovery

Brownfield commands can generate a `PromotionSource`:

```bash
kapro discover argo . --out kapro-connect --name checkout
kapro discover flux . --out kapro-connect --name checkout-flux
```

Review the generated source before adoption. Any unit marked low-confidence or
`needs-review` should be edited or removed before production use.
